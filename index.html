Hatching a Basilisk II Macintosh inside a Raspberry Pi Zero 2 W

Overview Hardware OS Setup Build Basilisk II Configuration Running macemu Fork

Overview

This guide walks you through building a compact, self-contained DIY Macintosh powered by a Raspberry Pi Zero 2 W running the Basilisk II 68k Macintosh emulator. The end result is a portable vintage-Mac experience in your pocket.

Basilisk II emulates a Motorola 680x0-based Macintosh, letting you run classic Mac OS (System 7 through Mac OS 8.1) on modern hardware. The Pi Zero 2 W is small, cheap, and just powerful enough to make the emulation usable.

⚠️ ROM & OS images: You will need a Macintosh ROM image and a Mac OS disk image. These are not provided here. You must obtain them legally — for example, by dumping the ROM from a Mac you own.

OS Setup (Raspberry Pi OS)

1. Flash Raspberry Pi OS Lite (32-bit)

Use Raspberry Pi Imager to flash Raspberry Pi OS Lite (32-bit) onto your microSD card. During imaging, use the advanced settings (⚙️) to:

  • Set hostname (e.g. basilisk)
  • Enable SSH
  • Configure Wi-Fi credentials
  • Set a username/password

2. First Boot & Update

Insert the card, power on, and SSH in (or attach a keyboard/monitor):

ssh pi@basilisk.local

Update the system:

sudo apt update && sudo apt full-upgrade -y
sudo reboot

3. Install Required Dependencies

Install the packages needed to build Basilisk II:

sudo apt install -y \
  git build-essential automake \
  libsdl2-dev libsdl2-ttf-dev \
  libgtk2.0-dev libglib2.0-dev \
  zlib1g-dev

4. (Optional) Set Up a Desktop Environment

Basilisk II can run headlessly with SDL2 output, but if you want a desktop you can install a lightweight one: 

sudo apt install -y xserver-xorg x11-xserver-utils openbox lxterminal
# Auto-start X on login:
echo 'startx' >> ~/.bash_profile

Build Basilisk II

1. Clone the macemu Fork

This guide uses the timothy-fuchs/macemu fork, which contains tweaks optimised for the Pi Zero 2 W. See the macemu Fork section for details. 

git clone https://github.com/timothy-fuchs/macemu.git
cd macemu/BasiliskII/src/Unix

2. Configure the Build

./autogen.sh
./configure \
  --enable-sdl-video \
  --enable-sdl-audio \
  --disable-vosf \
  --without-gtk
Note: --disable-vosf disables video-on-SIGSEGV optimisations that are not supported on ARM. --without-gtk avoids pulling in a full GTK stack and is fine when using the SDL interface.

3. Compile

make -j$(nproc)

Compilation will take several minutes on the Pi Zero 2 W. The resulting binary is BasiliskII in the current directory.

4. Install (Optional)

sudo make install

This copies the binary to /usr/local/bin/BasiliskII.

Configuration

Prepare ROM and Disk Images

Copy your Macintosh ROM file and a Mac OS disk image to the Pi — for example via scp: 

scp mac-rom.bin   pi@basilisk.local:/home/pi/mac/
scp macos-8.1.img pi@basilisk.local:/home/pi/mac/

Create the Preferences File

Basilisk II reads its settings from ~/.basilisk_ii_prefs. Create it with the following minimal configuration (adjust paths as needed): 

rom /home/pi/mac/mac-rom.bin
disk /home/pi/mac/macos-8.1.img

# RAM size in MB
ramsize 64

# CPU type: 4 = 68040
cpu 4
fpu false
nocdrom true

# Video: SDL window
screen win/800/600/0

# Keyboard / mouse
keyboardtype 5

# Network (optional — remove if not needed)
# ether slirp

Key Preferences Explained

  • rom — path to your Mac ROM image
  • disk — path to your Mac OS disk image (add more disk lines for extra volumes)
  • ramsize — emulated RAM in MB (32–256 MB recommended)
  • cpu 4 — emulate a 68040 processor
  • screen win/<W>/<H>/<depth> — SDL window resolution and colour depth (0 = current)

Running Basilisk II

Start the Emulator

BasiliskII --config /home/pi/.basilisk_ii_prefs

The SDL window will open and Mac OS will boot. Use your USB keyboard and mouse normally inside the window.

Autostart on Boot (Optional)

Create a systemd service so Basilisk II starts automatically when the Pi boots (headless SDL requires a display; adjust DISPLAY as needed): 

sudo tee /etc/systemd/system/basilisk.service <<EOF
[Unit]
Description=Basilisk II Mac Emulator
After=graphical.target

[Service]
User=pi
Environment=DISPLAY=:0
ExecStart=/usr/local/bin/BasiliskII --config /home/pi/.basilisk_ii_prefs
Restart=on-failure

[Install]
WantedBy=graphical.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable basilisk.service
sudo systemctl start  basilisk.service

Performance Tips

  • Use a fast microSD card — I/O speed directly affects disk image performance.
  • Lower the emulated resolution (e.g. 640/480) to reduce GPU load.
  • Set ramsize 32 if you hit memory pressure on the host.
  • Overclock the Pi Zero 2 W slightly via /boot/config.txt (e.g. arm_freq=1200) if your power supply can support it.

Troubleshooting

  • Black screen / immediate quit: verify the ROM path in ~/.basilisk_ii_prefs and ensure the file is a valid 512 KB or 1 MB Mac ROM dump.
  • SDL error: No available video device: make sure an X session is running, or try setting SDL_VIDEODRIVER=fbcon to use the framebuffer directly.
  • Very slow emulation: lower resolution, reduce ramsize, and close any background processes on the Pi.

macemu Fork

The build instructions above use a fork of the upstream cebix/macemu repository. The fork includes patches and configuration tweaks aimed at making Basilisk II run well on the Raspberry Pi Zero 2 W.
